1. What is DOM and what is the purpose of DOM?

Answer: DOM (Document Object Model) is an API for HTML. It represents the logical strucutre of a document and how a document can be accessed, read, and manipulated. Basically, all the interactive work of the website is done with DOM.

2. How will you select HTML elements using DOM? Name the DOM methods.?

Answer: I can select elements in a DOM using 5 selectors.

  • 1. document.getElementsByTagName()
  • 2. document.getElementsByClassName()
  • 3. document.getElementById()
  • 4. document.querySelector()
  • 5. document.querySelectorAll()

1. getElementsByTagName(HTML tag name/ h1/ p):The HTML tag name is first selected with the selector, then a variable is placed, that variable returns a HTML collection.

2. getElementsByClassName(HTML class name / .class): First i have to declare the class in the HTML tag and then i have to select the class with getElementsByClass(). then a variable is placed, that variable returns a HTML collection too.

3. document.getElementById(HTML ID nam / #id): This selector is the most used. It can be said that 50 percent of DOM selectors are used with it. With this you select a specific ID, which is declared ID in the HTML tag.

4. document.querySelector(): When you select by HTML id or class it shows you its first element.

5. document.querySelectorAll(): Give you a node list of match. then you can modify as your wish. [you can use array to access content of the list.]

3. What is event bubble?

Answer: An event bubble is a process in the DOM, which acts like a tree
When an event is triggered on a specific element, such as a mouse click or keyboard press, the event first handles itself and then works its way up to its parent, performing a task until it reaches the top-level element in the sequence.

4. What is a callback function and why will you use it?

Answer: Callback functions are very popular in JavaScript because it is an asynchronous language. This means, this function doesn't wait for another function or anything else to execute.


why will i use it: Suppose I download a file from the Internet, until it is downloaded, I want to do other work, it is not possible. So that's why the callback function is designed, for downloading, I can do my other tasks, it doesn't have to freeze.
